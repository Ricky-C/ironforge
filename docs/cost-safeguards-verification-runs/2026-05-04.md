## Phase 4: Deferred (not run)

The AWS Budgets threshold-detection and role-assumption path is
documented as outside the procedure's test surface
(`cost-safeguards.md` line 287). Phase 4 would push past that limit by
lowering the budget threshold below current spend to force the action
to fire, then verifying the AWS-internal path:

1. AWS Budgets evaluates spend > threshold
2. Assumes the executor role
3. Executor role attaches `IronforgeBudgetActionDeny` to target principals
4. `REVERSE_BUDGET_ACTION` recovery path reverses the attachment

Deferred because:

- AWS Budgets evaluation cadence is unbounded (3× per day per AWS docs; worst case ~8 hours)
- Deny-in-flight during the wait would block real CI / new service provisioning
- The AWS-side integration is documented as outside our test surface

Tracked in `docs/tech-debt.md` § "Phase 4: live threshold-trigger
verification of cost-safeguards" with concrete completion triggers.
